## Phase 2 — Інтерактивність після відповіді ✅
- [x] Кнопка **"Не згоден"** — inline форма в блоці пояснення, тільки при реальній неправильній відповіді (не при "далі"); Gemini переглядає оцінку, якщо змінює — рахунок перераховується; loading indicator під час очікування; форма з аргументом залишається після відповіді
- [x] Кнопка **"Поясни детальніше"** — задизейблена під час іспиту; після іспиту активується (стає помітною); розкриває inline-панель "Пояснення" у зеленому блоці з міні-чатом суто по цьому питанню; loading indicator і при завантаженні і в міні-чаті
- [x] Кнопка **"далі →"** — мутований текст-лінк внизу праворуч блоку питання (не есе); пропускає питання як неправильну відповідь, але без кнопки "Не згоден"
- [x] Після завершення іспиту чат залишається активним — учень може повернутись до будь-якого питання і продовжити розмову
- [x] Нічна тема і розмір шрифта — перенесені в налаштування (сегментовані контролери); тема підтримує режим "Авто" (`prefers-color-scheme`)
- [x] Кнопки "Новий іспит" / "Слабкі теми" / "Змінити предмет" у шапці — з'являються після завершення іспиту, зникають при старті нового
- [x] Вибрана кнопка в чаті залишається підсвіченою після кліку
- [x] Поле вводу — textarea з авторозміром 1–5 рядків, ліміт 1000 символів; Enter — надіслати, Shift+Enter — новий рядок
- [x] Персоналізація: бот звертається до учня на ім'я (`studentName` з config) у вітанні, завершенні іспиту і в Gemini feedback
- [x] Іконка розширення — 📚 emoji (PNG 16/48/128px), відображається і в тулбарі Chrome і в списку розширень
- [x] Секундомір у шапці — старт при початку іспиту, стоп при завершенні; час відображається у фінальному повідомленні

## Phase 2.5 — Gemini Nano (оптимізація запитів)
- [x] Флаг `useNano` в конфігурації — вмикає/вимикає Nano флоу
- [x] `Nano.checkAvailability()` — спільна утиліта перевірки доступності (`readily` / `after-download` / `no` / `unavailable`)
- [x] Вікно налаштувань: галка Nano перевіряє доступність при відкритті; якщо недоступний — блокується, знімається, записується `false`
- [ ] `"readily"` — модель вже є: отримати згоду користувача, тоді `create()`
- [ ] `"after-download"` — попередити про ~1.7GB, кнопка завантаження → `create()` з прогресом (`downloadprogress`)
- [ ] Промпт для Nano: предмет + питання + відповідь учня → `YES` (осмислена спроба) / `NO` (nonsense); вірш чи казка — `YES`
- [ ] Управління контекстним вікном (~6k токенів) — матрьошка пріоритетів:
  1. Системний промпт (предмет + всі питання іспиту + що це валідація)
  2. Поточне питання з відповіддю
  3. Останні 6 повідомлень чату (3+3)
  — якщо не вміщується — скорочуємо з кінця, системний промпт недоторканний
