# Study Buddy — Технічне завдання

## Ідея

Інтерактивна платформа для підготовки дитини до вступних іспитів у ліцей.
Дружина склала тести вручну в Gemini — ціль: перевести це в повноцінний інтерактив з LLM-генерацією питань, перевіркою відповідей і відстеженням прогресу.

**Важливий контекст:** це не навчання з нуля, а симуляція вступного іспиту. Учень вже знає матеріал — додаток перевіряє готовність. Питання генеруються в межах програми **1–(targetGrade-1) класів** — тобто того що учень вже мав вивчити.

Приклад: учень закінчує 7 клас, вступає в 8 клас ліцею → питання охоплюють програму 1–7 класів.

## Мета і ціль

**Мета** — дитина самостійно практикується: обирає предмет і тему, отримує питання рівня вступного іспиту, відповідає своїми словами, отримує зворотний зв'язок.

**Ціль MVP** — закрити підготовку конкретної дитини (математика + українська мова) до вступу в ліцей на рівні 8 класу.

**Ціль майбутнього** — масштабувати на будь-якого учня через просту конфігурацію: без оплат, без реклами, мінімум налаштувань.

## Стек

- **Chrome Extension** (Manifest V3) — side panel як основний UI; зберігається як режим
- **Web App / PWA** — той самий код, розгортається на GitHub/GitLab Pages; встановлюється на комп/телефон/планшет через браузер
- **Vanilla JS** — без фреймворків, без npm, без build-степу
- **Gemini API** — генерація питань + перевірка відповідей (потребує мережі)
- **Gemini Nano** (опційно) — локальна валідація відповідей; єдиний LLM що працює офлайн
- **localStorage** — прогрес, історія, налаштування (тема, шрифт, конфіг) між сесіями
- **sessionStorage** — питання поточної сесії
- Ніякого бекенду, ніякої БД

## Налаштування (config)

Зберігаються в `localStorage` (`sb_settings`). При першому запуску мігруються з `config/settings.json` (якщо є). Якщо файлу немає — поля порожні, відкривається модальне вікно налаштувань.

```json
{
  "apiKey": "",
  "studentName": "Ім'я",
  "targetGrade": 8,
  "questionsPerSession": 7,
  "model": "gemini-2.5-flash-lite-preview",
  "useNano": false
}
```

`targetGrade` — клас до якого вступає учень. Питання генеруються в межах програми **1–(targetGrade-1) класів** — тобто того що учень вже мав вивчити. Також фільтрує теми у subjects.

`useNano` — використовувати Gemini Nano для валідації відповідей. Автоматично скидається в `false` якщо Nano недоступний на пристрої.

## Флоу користувача

1. Відкриває side panel — бачить привітання з класом вступу
2. Якщо `apiKey` порожній — системне попередження + автоматично відкривається вікно налаштувань
3. У шапці — кнопка ⚙️ налаштувань, перемикач розміру шрифта (малий/середній/великий, `sb_font` в localStorage), секундомір (видимий під час іспиту) і перемикач теми (`sb_theme`)
4. Обирає предмет через кнопки в чаті; вибрана кнопка залишається підсвіченою, решта блокуються
5. Якщо `studentName` заповнено — бот звертається на ім'я у вітанні, завершенні, і Gemini feedback (checkAnswer, evaluateEssay, disputeAnswer, freeChat)
4. Gemini отримує відфільтрований список тем (`grade <= targetGrade-1`) і самостійно формує іспит: вибирає теми рівномірно з різних розділів, питання перемішані
5. Для української — `questionsPerSession-1` мовних питань + есе останнім
6. Питання задаються по одному з прогрес-баром, учень відповідає у textarea (авторозмір 1–5 рядків, ліміт 1000 символів; Enter — надіслати, Shift+Enter — новий рядок)
7. Після кожної оцінки — кнопки **"Не згоден"** і **"Поясни детальніше"** (Phase 2)
   - "Не згоден" → inline форма для аргументу; після відповіді форма залишається заблокованою (аргумент дитини зберігається)
   - "Поясни детальніше" → inline панель у зеленому блоці з поясненням і міні-чатом по питанню
8. Після завершення іспиту — результат і кнопки "Новий іспит" / "Змінити предмет" у шапці
9. Чат залишається активним — учень може продовжити розмову по будь-якому питанню (Phase 2)
10. Результат зберігається в localStorage

## Gemini промпти

**Генерація іспиту:**
```
Ти екзаменатор, який приймає вступний іспит до {targetGrade} класу ліцею.
Предмет: {subjectName}.
Доступні теми (програма 1–{targetGrade-1} класів): {список тем}.

Склади іспит з {count} відкритих питань різної складності.
Вибери теми рівномірно з різних розділів, питання перемішай.

Відповідь — лише JSON масив:
[{ "type": "question", "question": "...", "answer": "...", "hint": "...", "topic": "назва теми" }]
```

**Генерація іспиту з есе (українська мова):**
```
...те саме, але останнє завдання — есе:
{ "type": "essay", "question": "Напиши есе (10–15 речень) на тему: ...", "answer": "", "hint": "", "topic": "Есе" }
```

**Перевірка відповіді:**
```
Питання: {question}
Правильна відповідь: {expectedAnswer}
Відповідь учня: {studentAnswer}

Відповідь — лише JSON: { "correct": true, "explanation": "..." }
Пояснення українською, коротко і доброзичливо — учень закінчує {targetGrade-1} клас.
```

**Оцінювання есе:**
```
Завдання: {essayPrompt}
Текст учня: {studentEssay}

Оціни: відповідність темі, структура, грамотність, стиль.
Відповідь — лише JSON: { "correct": true, "explanation": "..." }
Доброзичливо, з конкретними порадами.
```

**"Не згоден" (Phase 2):**
```
Питання: {question} | Правильна відповідь: {expectedAnswer}
Відповідь учня: {studentAnswer} | Попередня оцінка: {correct/incorrect}
Аргумент учня: {dispute}

Розглянь аргумент. Відповідь — лише JSON: { "correct": true, "explanation": "...", "revised": true/false }
```

### Web / PWA режим (заплановано)

Ціль — один кодовий базис, два способи використання:
- **Chrome Extension** — side panel, як зараз
- **Web App** — розгортається на GitHub/GitLab Pages, відкривається в будь-якому браузері
- **PWA** — встановлюється через браузер на комп/телефон/планшет як нативний додаток

Технічно:
- `getResourceURL(path)` — хелпер: `chrome.runtime.getURL` в extension, відносний шлях у веб
- `index.html` — точка входу для веб (той самий UI)
- `site.webmanifest` — PWA маніфест (окремий від extension manifest.json)
- `config/settings.json` — не комітити, тільки `settings.template.json`; ключ вводиться через UI

### Офлайн-режим (заплановано)

- `navigator.onLine` + події `offline`/`online` — детекція стану мережі
- При втраті з'єднання — системний меседж в чаті, блокування викликів Gemini API
- При відновленні — повідомлення в чаті
- Виняток: якщо увімкнено Gemini Nano — перевірка відповідей доступна офлайн (генерація питань потребує мережі)
